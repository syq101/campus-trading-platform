# 团队协作实现指南

## Git团队协作步骤

### 1. 创建GitHub仓库
1. 访问 https://github.com/new 创建新仓库
2. 仓库名称建议：`campus-trading-platform`
3. 选择 Public 或 Private
4. **不要**勾选 "Initialize this repository with a README"
5. 点击 "Create repository"

### 2. 关联远程仓库

```bash
# 在当前目录执行
git remote add origin https://github.com/你的用户名/campus-trading-platform.git
```

### 3. 团队协作工作流

#### 方式一：按模块分工（推荐）

**成员1 - 后端开发**：
```bash
git checkout -b feature/backend
# 修改 server.js 和 database.js
git add server.js database.js
git commit -m "实现后端API和数据库"
git push origin feature/backend
```

**成员2 - 买家端开发**：
```bash
git checkout -b feature/buyer
# 修改 public/buyer.html
git add public/buyer.html
git commit -m "实现买家端页面"
git push origin feature/buyer
```

**成员3 - 卖家端开发**：
```bash
git checkout -b feature/seller
# 修改 public/seller.html
git add public/seller.html
git commit -m "实现卖家端页面"
git push origin feature/seller
```

**成员4 - 文档和测试**：
```bash
git checkout -b feature/docs
# 修改 README.md
git add README.md
git commit -m "完善项目文档"
git push origin feature/docs
```

### 4. 合并分支

每个成员完成后，在GitHub上创建 Pull Request，团队审查后合并到 main 分支。

### 5. 初始提交（如果需要）

如果想要先提交当前代码：

```bash
git add .
git commit -m "初始提交：校园二手交易平台"
git branch -M main
git push -u origin main
```

## 快速测试验收

### 测试验收标准1

1. 打开卖家端 http://localhost:3000/seller.html
2. 测试添加、编辑、删除商品
3. 刷新页面验证数据持久化

### 测试验收标准2

1. 打开两个浏览器窗口
2. 窗口A：买家端 http://localhost:3000/buyer.html
3. 窗口B：卖家端 http://localhost:3000/seller.html
4. 在买家端点击"立即购买"
5. 观察卖家端是否收到弹窗通知

### 测试验收标准3

查看GitHub仓库的Contributors页面，确认每个成员都有提交记录。
